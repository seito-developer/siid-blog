# 記事入稿ガイドライン

microCMS で記事を作成・編集する際のルール。SEO コンサル指摘（2026-07）を受けて策定（Issue #41）。

## 見出し

- **本文の見出しは H2 始まり**にする。H1 は使わない（記事タイトルがテンプレート側で H1 として出力されるため、本文に H1 があると1ページ複数 H1 になる）
- 見出しレベルは飛ばさない（H2 の下は H3。H2 の直下に H4 を使わない）

## スラッグ（コンテンツ ID）

- **内容が分かる英語のケバブケース**にする（例: `sql-beginner`, `laravel-sail-commands`）
- microCMS が自動生成するランダム文字列（例: `qvm46tro91`）のまま公開しない
- 公開後のスラッグ変更は URL 変更（リンク切れ・評価リセット）になるため行わない

## リンク

- リンクの **href に注記や説明文を混ぜない**（例: `href="https://mid-works.com（運営元の公式サービスサイト）"` は無効リンクになる。注記はリンクの外に書く）
- **ローカル開発 URL**（`http://127.0.0.1:8000` など）や**例示用ドメイン**（`evil.com` など）はリンクにせず、リッチエディタの「コード」表記にする

## 画像

- 画像には**内容を表す alt テキスト**を入力する（画像検索・アクセシビリティ対策。装飾目的の画像のみ空で可）
- 入稿画像は**幅 2000px 以下**を目安に縮小してからアップロードする（原寸のまま上げると転送量が過大になる）

## 説明文（excerpt）

- microCMS に説明文フィールド（`excerpt`）が用意されている場合は、**記事の要約（80〜120文字）**を入力する。meta description / SNS シェア文に優先使用される（未入力時は本文冒頭からの自動生成にフォールバック）
