# SiiD BLOG 改善施策提案書

作成日: 2026-07-17
作成方法: 競合サイト実地調査（ブラウザ閲覧）＋ 3視点（Webディレクター / デザイナー / SEOコンサルタント）のエージェント分析を統合

## 目的

SiiD BLOG（https://blog.bug-fix.org/）の目的は **SEO流入 → 回遊 → スクール受講CV（無料個別面談・LINE登録）** の実現。本書は競合分析に基づき、少人数運営でも回る改善施策を優先度付きで提案する。

---

## 1. 現状分析（自サイトの課題）

### 1-1. 構造的課題の要約

SiiD BLOG は「入口（172本のSEO記事）」はあるが、**「経路（回遊）」と「出口の面積（CV導線）」が極端に細い**。

| 領域 | 現状 | 問題 |
|---|---|---|
| 回遊 | 関連記事・前後記事・人気記事・グローバルナビ・カテゴリ導線がすべて無い | SEO着地後の行き先が「離脱」か「記事末CTA」の二択。172記事が相互リンクせず、PV/セッション・内部リンク評価を取りこぼし |
| CV導線 | 記事末のテキストリンク（面談/LINE/YouTube/X）＋画像バナー1枚のみ | 記事を最後まで読んだ人にしか当たらない一点集中。途中離脱層を全て逃す |
| シェア | ボタン無し | 拡散・被リンク獲得の機会損失 |
| TOP | テキスト見出し＋検索＋新着グリッドのみ | メディアとしての顔が無く、注目記事・カテゴリ導線・スクール認知の装置が無い |
| E-E-A-T | 著者は名前表示のみ（description フィールドは保有済みだが未表示） | キャリア・転職領域（YMYL寄り）で著者の信頼性シグナルが弱い |

### 1-2. 維持すべき強み

- **技術SEO基盤は良好**: sitemap / robots / canonical / OGP / JSON-LD（WebSite, Article, BreadcrumbList）/ パンくず / ISR が整備済み
- **デザインのトーン**: `#214a4a`（深緑）× `#F4F4F4` の落ち着いた配色は、競合の赤・オレンジの「圧の強い」商業感に対する差別化資産
- **記事カード（`article.tsx`）の完成度**: ホバー演出・カテゴリバッジ・メタ情報が揃っており、関連記事・注目記事にそのまま流用可能
- **著者資産**: セイト先生（YouTube 12万人）は強力な E-E-A-T シグナル。データはあるのに UI に出ていない

### 1-3. 発見された軽微な不具合

- `web/src/app/page.tsx` にデバッグ用の `Page: {page}` 見出しが本番に残存（要削除）
- 記事タイトルの `<title>` に `| SiiD BLOG` サフィックスが無い（TOP・カテゴリには有り＝不統一）

### 2-2. 競合から学ぶべきポイント

1. **「読み終わりに次の一歩」が必ずある**: 関連記事6件＋文脈CTAで、読了ユーザーを離脱させない
2. **著者ボックスによる E-E-A-T**: 侍の「この記事を書いた人」は顔写真＋経歴で信頼を担保し、CVの心理的ハードルを下げる
3. **ヘッダー常設CTA**: どのページ・どのスクロール位置からでもCVできる「出口の常設」
4. **UTM付きCTAで計測**: 侍は全CTAにUTMを付与し、効く位置・記事を特定して改善を回している
5. **卒業生の before/after 訴求**: 「経理→Webエンジニア」のような職種変化カードは訴求力が高い（ただし制作コスト高、将来課題）

### 2-3. 真似すべきでないもの（根拠付き）

| 競合施策 | 真似すべきでない理由 | SiiDの代替策 |
|---|---|---|
| 訪問直後のポップアップモーダル（RUNTEQ） | Googleが評価を下げる「煩わしいインタースティシャル」に該当しうる。SEO流入が生命線のSiiDにはリスク直撃。価値提供前の勧誘は直帰を誘発 | 読了に応じた追従バー・本文途中CTAで「読ませてから当てる」 |
| キャンペーンバナー常時更新（RUNTEQ） | 販促体力と更新リソースが前提。少人数運営では鮮度維持できず、古い訴求が残ると不信を招く | 追従は「無料相談」1本に固定。キャンペーン時のみ差し込み |
| 記事内CTAの物量配置（侍） | 超長文×高トラフィックでのA/B最適化の産物。密度だけ真似ると「押し付けがましいスクール広告」化しブランド毀損 | 本文途中1〜2点＋記事末1＋追従1の計3〜4点から開始し、計測して調整 |
| インタビュー・動画の量産 | 取材・制作リソースを大量消費し少人数では持続不可能 | 優先度を下げる。まず既存172記事の導線改善で成果を出し、余力が出たら質の高い1本から |

## 4. コンテンツ戦略（SEOコンサル視点の要約)

### キーワード2軸設計

- **軸A（定番・転職ワード）**: 「プログラミングスクール 比較/評判」「エンジニア転職 未経験」等。CVに近いが競合激戦。SiiD の評判・体験談系で差別化
- **軸B（時流・AIワード）**: 「AI駆動開発」「Claude Code 使い方」「生成AI プログラミング学習」等。**「AIプログラミングスクール」を名乗る SiiD の主戦場**。競合の蓄積が浅く後発でも上位を狙え、検索需要が急拡大中

**導線設計**: 軸Bで広く集客 → 内部リンクで軸Aの比較・評判記事へ誘導 → 面談CV。この意味でも P1 の内部リンク施策（関連記事・contextual リンク）が土台になる。

### CV設計の考え方

ブログ読者は検討初期の低温度層が多いため、**LINE登録をマイクロコンバージョン**として第一目標に置き、LINE内で無料面談へ引き上げる二段構えを推奨。

| 検索意図 | CV距離 | 適切なCTA |
|---|---|---|
| Know（「Claude Codeとは」） | 遠 | LINE登録（弱CTA）＋関連記事で回遊 |
| Know-学習（「Rails 環境構築」） | 中 | 「独学の限界→スクール」文脈CTA |
| Compare（「スクール比較」） | 近 | 無料面談（強CTA） |
| 指名（「SiiD 評判」） | 最近 | 無料面談・申込 |

## 5. デザイン方針（デザイナー視点の要約）

**コンセプト: 「キャンペーンで殴らない、信頼できる技術メディアの静かな権威性」**

- 競合の赤・オレンジ×物量CTAに対し、深緑 `#214a4a` × ティール `#289B8F` の落ち着いたトーンで差別化
- ヒーローは**カルーセル**とし、スライドは「① サービスサイト導線 / ② 給付金解説記事 / ③ キャンペーン（任意）」の固定2枚＋随時1枚で構成。RUNTEQ 型の「常時キャンペーン更新」はせず、更新頻度の低い恒常導線を主体にして少人数運営でも鮮度が劣化しない設計にする
- 注目記事は microCMS のフラグ or カテゴリで自動抽出し「作って放置」で回る設計
- シェアボタンは SNS ブランドカラーを敷き詰めず、モノトーン＋ホバーティールで世界観を維持

**アクセシビリティ注意**: ティール `#289B8F` ×白文字はコントラスト比約3.0で WCAG AA（4.5:1）未達。**重要テキストのボタンは `#214a4a`（約8:1）を使用**し、ティールは大きい文字・装飾・ボーダーに限定する。TOP再構成時は記事カードの見出しレベルを h2→h3 に調整して階層逆転を防ぐ。

新規コンポーネント仕様（GlobalHeader / HeroSection / FeaturedArticles / CategoryNav / CtaBand / ArticleToc / ShareButtons / AuthorCard / ArticleCtaCard / RelatedArticles）の詳細はワイヤーフレーム（§6）を参照。

## 付記

- 本提案の施策は原則 **microCMS のスキーマ変更不要**（関連記事・ナビ・著者カード・JSON-LD はすべて既存フィールドとクエリで実現可能）。例外は TOP「注目記事」のフラグフィールド追加（任意）
- 本文レンダリングに関わる実装（目次の id 付与等）は必ず `sanitize-article-html.tsx` のサニタイズを通すこと
